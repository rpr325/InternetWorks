How the Internet Works: Hypertext Transfer Protocols



HTTP

As the user interacts elements on the web a series of ‘get’ requests are sent for the contents that appear on the page. For example if the user goes to http://espn.com the browser will create and send a series of HTTP requests in order to get the information needed to render the webpage. The requests received are called response requests.

Hypertext Transfer Protocol (HTTP) is a protocol – or a prescribed order and syntax for presenting information – used for transferring data over a network.

An HTTP request is text formatted in HTTP protocol.

The get request is generated by the user’s browser and sent across the internet. Since the internet is an open system anyone can view the get or response requests. This is not a secure form of transmission across the web, especially when sending sensitive data.

HTTP

Sending secure get and response request over the internet require Hypertext Transfer Protocol Secure (HTTPS). HTTPS uses TLS (or SSL) to encrypt HTTP requests and responses. Instead of seeing the request data an external viewer would see a string of random text.

HTTP is a public protocol and thus readable to outside viewers. However, with HTTPS security is prioritized to ensure transmission data is kept private. This is facilitated by public key encryption. With public key encryption there are two keys, a public key and private key. The public key is shared with client devices via the server’s SSL certificate. This allows for the devices to agree on new keys, called session keys, to encrypt subsequent communications.
