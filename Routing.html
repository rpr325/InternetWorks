How the Internet Works: IP Addresses, Packets and Routing

Data transmitted over the internet travels in an indirect fashion. This is the nature of a distributed packet network to ensure fault tolerance and redundancy. Users navigate to connected devices using the IP address to request information. The information request and delivery are managed by routers to ensure information moves between the user and destination devices. Packets are the means in which portions of the information are transmitted. These three parts ensure a reliable distributed communication network that can scale as more users and devices are added.

IP Adresses

When a user inputs a domain name into the browser on their computer it kicks off a journey of data across the internet to the destination device. The first action is for the domain name is resolved through the domain name system. This is where a domain name server provides the user device with the IP Address for the destination device on the internet.

Routing

Routing is the process of moving packets across the network along the most efficient route. This ensures that the internet is fault tolerant. This feature is key to the distributed nature of the internet.

Packets

Information is transmitted through the web using packets. The internet was designed as a distributed packet network. Packets are bits of data sent in clusters from source to destination device.
